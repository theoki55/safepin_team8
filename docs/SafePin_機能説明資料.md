# SafePin 機能説明資料

**在宅避難の見えない孤立を、地図でつなぐピン立てマップ**

- バージョン: 1.0.0+1
- 対応地域（コミュニティ）: **下目黒地区**（下目黒4・5・6丁目）／ **東京都目黒区** ／ **千葉県流山市**
- プラットフォーム: Flutter Web（スマホ・PC ブラウザで動作 / Android アプリ化も可能）
- 地図: OpenStreetMap（APIキー不要）
- バックエンド: Firebase（Cloud Firestore / Storage）

---

## 1. このアプリが解決したい課題

大規模災害時、避難所ではなく**自宅にとどまる「在宅避難者」**は、支援の網から漏れやすく、
「誰が・どこで・何に困っているか」が外から見えにくいという問題があります。

SafePin は、地域住民が地図上に「ピン」を立て合うことで、

- **助けてほしいこと（NEED）**
- **手伝えること（OFFER）**
- **地域の状況（INFO）**

を可視化し、住民同士・自治会が**顔の見える範囲で助け合う**ことを目指す地域限定のマップアプリです。

---

## 2. 全体構成（3つの画面）

アプリ下部のタブで3つの画面を切り替えます。

| タブ | 画面 | 役割 |
|------|------|------|
| 🗺️ 地図 | マップ画面 | ピンを地図上に表示・投稿・詳細確認 |
| 📋 一覧 | リスト画面 | ピンを新着順／緊急度順で一覧表示 |
| ⚙️ 設定 | 設定画面 | 対象地域切替・モード切替・投稿者名・管理者モード・地域資源管理・データ管理 |

---

## 3. ピンの種類（NEED / OFFER / INFO）

投稿は3種類に分類され、色とアイコンで一目で区別できます。

| 種別 | 色 | 内容 | 例 |
|------|-----|------|-----|
| **NEED（助けて）** | 🔴 赤 | SOS・安否・不足物資など、必要な支援 | 「水が不足しています」 |
| **OFFER（手伝える）** | 🔵 青 | 提供できる物資・スキル・人手 | 「モバイルバッテリー貸せます」 |
| **INFO（地域情報）** | 🟢 緑 | 給水所・通行止め・炊き出しなど地域情報 | 「〇〇公園で給水中」 |

各ピンには以下の情報を付けられます。

- **タイトル / コメント（詳細）**
- **緊急度**：高（赤）／中（黄）／低（緑）のトリアージ
- **写真・ファイル添付**：カメラ撮影・写真選択・ファイル添付に対応
- **投稿者名**（任意・未入力なら「匿名」）

---

## 3-A. 地域資源（RESOURCE）

NEED / OFFER / INFO が**その時々の一時的な投稿**であるのに対し、
**地域資源（RESOURCE）**は、消火器・AED・防災倉庫といった
**平時から地域に常設されている「恒久的な設備」**を地図に登録・共有する機能です。
災害時だけでなく**平時から**「近くにどんな防災リソースがあるか」を把握できます。

### 資源のカテゴリ（第4系統）
既存3ピン（赤=NEED／青=OFFER／緑=INFO）と色が衝突しないよう、
資源は**紫〜茶・水色系の「第4系統」**の色とアイコンで表示します。

| カテゴリ | 内容 |
|----------|------|
| 🧯 街頭消火器 | 街頭に設置された消火器の位置 |
| 🧱 土のう置き場 | 浸水対策の土のう・水のうの置き場 |
| ❤️ AED | AED（自動体外式除細動器）の設置場所 |
| 🏭 防災倉庫 | 防災資機材を保管する倉庫 |
| 💧 井戸 | 生活用水として使える井戸 |
| 🚰 給水拠点 | 応急給水栓・給水拠点 |
| 👥 一時集合場所 | 発災時に近隣で集まる一時集合場所 |

### ピンとの違い・特徴
- **正確な位置で常設表示**：資源は公共・共用の設備のため、位置ぼかし（→6章）は行わず、
  地図上に**正確な地点**を紫の角丸マーカーで表示します。
- **登録できるのは管理者のみ**：地域資源の登録・削除は**管理者モード**（→7-2）でのみ可能です
  （一般ユーザーは閲覧のみ）。
- 各資源には、**名称・カテゴリ・住所・メモ・管理者（管理団体）・最終点検日・利用可否**などを設定できます。
- 資源の登録・削除は**監査ログ**に記録されます。

### 表示の切り替え
- 地図左下の**「資源」トグル**で、資源レイヤーの表示ON/OFFと**登録件数**を確認できます。
- 凡例にも「地域資源」の色を表示します。

### CSV 一括登録
自治会が保有する資源リストを**CSVでまとめて取り込め**ます（設定画面 → 地域資源をCSV一括登録）。

- 必須列：`category`（カテゴリ）／`name`（名称）／`lat`（緯度）／`lng`（経度）
- 任意列：`address`（住所）／`note`（メモ）／`managed_by`（管理団体）／`last_inspected`（最終点検日）／`available`（利用可否）
- ヘッダは**順不同**。`category` は enum キー（例：`fireExtinguisher`）でも日本語ラベル（例：`街頭消火器`）でも指定可能。
- 取り込み時に**必須列・数値/範囲・対象区域**をチェックし、問題行は警告として提示します。

> ※ 地域資源も**所属コミュニティ（communityId）ごとに分離**されます（→8-2）。
> 登録した地域の資源のみが、その地域を選択中に表示されます。

## 4. 対応ステータス（状況の進行管理）

ピンには「今どういう状況か」を示すステータスがあり、対応の進捗を共有できます。

- **NEED / OFFER**：未確認 → 現地確認済 → 支援調整中 → 対応中 → 対応済
- **INFO**：未確認 → 現地確認済

ステータスはステッパー（段階表示）で視覚化され、投稿者本人または管理者が変更できます。

## 5. 主要機能一覧

### 5-1. 地図でのピン表示・投稿
- OpenStreetMap 上にピンを表示。地図を**長押し**するとその場所にピンを立てられます。
- 上部に **NEED / OFFER / INFO の件数と緊急件数** を常時表示。
- 「現在地」ボタンで自分の位置へ移動、「ピンを立てる」ボタンで新規投稿。

### 5-2. 位置の指定方法
投稿時、位置は2通りで指定できます。
- **現在地から取得**（GPS）
- **地図で指定**：中央固定ピンを目的地に合わせる方式

### 5-3. 一覧表示・並べ替え
- ピンをカード形式で一覧表示。
- **新着順 / 緊急度順** で並べ替え可能。

### 5-4. 絞り込み（フィルター）
- **種別**（NEED/OFFER/INFO）、**ステータス**で絞り込み。
- 「対応済を隠す」オプション。
- モード（平時／災害）に応じた**おすすめフィルター**を自動適用。

### 5-5. 平時モード / 災害モード
- **災害モード**では、SOS・安否・物資ニーズを優先的に表示（上部に赤い警告バナー）。
- モードは設定画面から切り替え、端末に保存されます。

## 6. プライバシー保護：位置ぼかし機能

在宅避難者の安全に配慮した、SafePin の重要な特徴です。

- **NEED / OFFER のピン**は自宅など生活拠点と結びつきやすいため、
  正確な座標ではなく**約150m四方のマスの中心**に丸めて表示します（防犯上のリスク低減）。
- 地図上には「このあたり」を示す**円**を描画し、正確な地点ではないことを利用者に伝えます。
- **INFO（給水所・通行止め等）**は地点そのものが有用なため、正確な位置で表示します。
- 元データ（Firestore 上の座標）は変更せず、**表示時のみ**丸める設計です。

## 7. コミュニティ運用機能（ステップB）

住民運用での荒れ・混乱を防ぐための仕組みです。

### 7-1. ① 通報・自動非表示
- 不適切な投稿は誰でも「通報」できます。
- **3件**の通報が集まった投稿は**自動的に非表示**になります（一般ユーザーには見えなくなる）。
- 管理者は非表示の投稿も確認でき、必要に応じて**非表示を解除**できます。

### 7-2. ② 管理者モード（合言葉方式）
- 自治会役員など運用者は、**合言葉**を入力することで管理者モードに切り替えられます。
- 管理者は、**すべての投稿の編集・削除・ステータス変更・非表示解除**が可能。
- 合言葉は SHA-256 ハッシュで照合し、平文はアプリに保持しません。
- 管理者セッションには**有効期限（既定60分）**があり、期限切れで自動解除。共有端末での付けっぱなしを防ぎます。
- 「いつ・誰が・何をしたか」を**監査ログ**（管理者ログイン／資源登録・削除／パスワード変更 等）に記録します。
- 合言葉は**コミュニティ（地域）ごとに個別**で、設定画面から**変更**もできます（→ 8-2）。
- 設定画面からいつでも管理者モードを解除できます。

### 7-3. ④ 投稿の信頼度シグナル
投稿に対して住民が反応を付けられます。

| 反応 | 意味 | 効果 |
|------|------|------|
| 🟠 現地確認済 | 実際に見て確認した | **2人**が押すと未確認→現地確認済へ自動昇格 |
| 🟢 役に立った | 有用な情報だった | 信頼度の指標 |
| 🟤 古い情報 | もう古い可能性がある | 「役に立った」を上回り**3件**超で「古い可能性」警告を表示 |

- 一覧カードにも「古い可能性」バッジを表示し、鮮度を判断しやすくします。

## 8. 対応地域（コミュニティ）と対象区域

### 8-1. サービス対象区域の制限（機能⑥）

地域限定の運用のため、投稿できる区域をガイドします。判定方法はコミュニティの種別によって異なります。

**丁目ポリゴン型（例：下目黒地区）**
- 対象区域は **下目黒4・5・6丁目**（e-Stat 令和2年国勢調査の丁目境界データを使用）。
- 地図上に**区域境界を緑のポリゴン**で表示。
- 投稿時、位置が**区域内か区域外か**をリアルタイムで判定・表示：
  - 区域内：緑のインジケーター／赤いピン
  - 区域外：オレンジの警告インジケーター／×付きオレンジピン
- 区域外でも**投稿はブロックせず「警告のみ」**（区域内投稿を推奨する運用）。

**市区町村型（例：目黒区・流山市）**
- 市区町村全体を対象とするため、**区域境界ポリゴンや区域外警告は表示しません**。
- 市内のどこでも投稿でき、細かな丁目単位の判定は行いません。

> ※ 国勢調査の丁目境界は、住居表示で用いる実際の境界と多少異なる場合があるため、
> あくまで目安として利用します。
> 出典：e-Stat 令和2年国勢調査 町丁・字等別境界データ

---

### 8-2. 複数コミュニティ対応（自治会・地域の切り替え）

SafePin は**複数の地域（コミュニティ）**を1つのアプリで運用できます。
現在は **下目黒地区／東京都目黒区／千葉県流山市** の3コミュニティに対応しています。

#### データの分離
- ピン・地域資源はそれぞれ**所属コミュニティ（communityId）**を持ちます。
- 表示・件数・一覧・地図は**選択中のコミュニティのデータだけ**に絞り込まれ、
  **異なる地域のデータが混ざることはありません**（例：目黒区と流山市は互いに見えない）。
- 新しく投稿・登録したデータは、自動的に**その時選択中のコミュニティ**に紐づきます。

#### 地域の切り替え方
優先順位は次のとおりです。

1. **URL パラメータ `?c=<id>`**（最優先）
   - 例：`https://…/?c=nagareyama` で流山市を直接開く。
   - 自治会ごとに**専用URL**を配布すれば、住民は迷わず自分の地域を開けます。
   - 有効な id：`shimomeguro`（下目黒地区）／`meguro`（目黒区）／`nagareyama`（流山市）
2. **端末に保存された選択**（前回選んだ地域）
3. **既定のコミュニティ**（下目黒地区）

さらに、**設定画面の「対象地域」**から手動で切り替えることもできます。
切り替え時は、地図が自動的にその地域の中心・ズームへ移動します。

#### 地域ごとの独立設定
コミュニティごとに以下を個別に持ちます。

| 項目 | 内容 |
|------|------|
| 名称 | 下目黒地区／東京都目黒区／千葉県流山市 |
| 地図の中心・ズーム | 地域に最適化した初期表示 |
| 対象区域の判定方式 | 丁目ポリゴン型 or 市区町村型（8-1） |
| 管理者の合言葉 | **コミュニティ別**（設定画面で変更可能） |

- **ヘッダに現在の地域名バッジ**を常時表示（📍＝丁目ポリゴン型／🏙＝市区町村型のアイコン）。
  バッジをタップすると設定画面の地域切替へ移動します。
- 地域を切り替えると、**セキュリティのため管理者モードは自動的に解除**されます
  （別地域の権限をそのまま持ち越さない）。
- 管理パスワードは地域ごとに独立して保存され、**一方の地域で変更しても他地域には影響しません**。

> ※ 既存の下目黒地区のデータは、そのまま「下目黒地区」コミュニティに引き継がれます。

## 9. データ管理機能（設定画面）

- **投稿者名の設定**：次回以降の投稿にデフォルトで入る名前。
- **複数選択して削除**：自分の投稿（管理者は全投稿）をまとめて削除。
- **すべての投稿を削除**：自分の投稿（管理者は全投稿）を一括削除。
- **CSV / JSON インポート・エクスポート**：ピンデータを一括取り込み・書き出し
  （初期データ投入やバックアップに利用）。
- **地域資源の管理（管理者）**：地域資源の登録・CSV一括登録（→ 3-A）。

---

## 10. 匿名性の仕組み

- ログイン不要。アプリ初回起動時に**端末ごとの匿名ID（UUID）**を自動発行し、端末内に保存します。
- このIDにより、「自分が投稿したピンかどうか」を判定し、**本人だけが自分の投稿を編集・削除**できます。
- メールアドレスや個人アカウントは一切不要で、**気軽かつ匿名で参加**できます。

## 11. 技術構成（概要）

| 領域 | 採用技術 |
|------|----------|
| フレームワーク | Flutter 3.35.4 / Dart 3.9.2 |
| 地図 | flutter_map + OpenStreetMap（APIキー不要） |
| バックエンド | Firebase（Cloud Firestore：リアルタイム同期 / Storage：添付） |
| 状態管理 | Provider |
| ローカル保存 | Hive（文書DB）/ shared_preferences（設定） |
| 位置情報 | geolocator |
| 匿名認証 | 端末発行の自前 UUID（外部認証サービス不使用） |

- **リアルタイム同期**：Firestore により、誰かがピンを立てると他の端末にも即時反映。
- **オフライン耐性**：ローカル保存を併用し、通信が不安定な災害時にも動作。
