# SafePin 機能説明資料

**在宅避難の見えない孤立を、地図でつなぐピン立てマップ**

- バージョン: 1.0.0+1
- 対象地域（実証運用）: 東京都目黒区 **下目黒4・5・6丁目**
- プラットフォーム: Flutter Web（スマホ・PC ブラウザで動作 / Android アプリ化も可能）
- 地図: OpenStreetMap（APIキー不要）
- バックエンド: Firebase（Cloud Firestore / Storage）

---

## 1. このアプリが解決したい課題

大規模災害時、避難所ではなく**自宅にとどまる「在宅避難者」**は、支援の網から漏れやすく、
「誰が・どこで・何に困っているか」が外から見えにくいという問題があります。

SafePin は、地域住民が地図上に「ピン」を立て合うことで、

- **助けてほしいこと（NEED）**
- **手伝えること（OFFER）**
- **地域の状況（INFO）**

を可視化し、住民同士・自治会が**顔の見える範囲で助け合う**ことを目指す地域限定のマップアプリです。

---

## 2. 全体構成（3つの画面）

アプリ下部のタブで3つの画面を切り替えます。

| タブ | 画面 | 役割 |
|------|------|------|
| 🗺️ 地図 | マップ画面 | ピンを地図上に表示・投稿・詳細確認 |
| 📋 一覧 | リスト画面 | ピンを新着順／緊急度順で一覧表示 |
| ⚙️ 設定 | 設定画面 | モード切替・投稿者名・管理者モード・データ管理 |

---

## 3. ピンの種類（NEED / OFFER / INFO）

投稿は3種類に分類され、色とアイコンで一目で区別できます。

| 種別 | 色 | 内容 | 例 |
|------|-----|------|-----|
| **NEED（助けて）** | 🔴 赤 | SOS・安否・不足物資など、必要な支援 | 「水が不足しています」 |
| **OFFER（手伝える）** | 🔵 青 | 提供できる物資・スキル・人手 | 「モバイルバッテリー貸せます」 |
| **INFO（地域情報）** | 🟢 緑 | 給水所・通行止め・炊き出しなど地域情報 | 「〇〇公園で給水中」 |

各ピンには以下の情報を付けられます。

- **タイトル / コメント（詳細）**
- **緊急度**：高（赤）／中（黄）／低（緑）のトリアージ
- **写真・ファイル添付**：カメラ撮影・写真選択・ファイル添付に対応
- **投稿者名**（任意・未入力なら「匿名」）

---

## 4. 対応ステータス（状況の進行管理）

ピンには「今どういう状況か」を示すステータスがあり、対応の進捗を共有できます。

- **NEED / OFFER**：未確認 → 現地確認済 → 支援調整中 → 対応中 → 対応済
- **INFO**：未確認 → 現地確認済

ステータスはステッパー（段階表示）で視覚化され、投稿者本人または管理者が変更できます。

---

## 5. 主要機能一覧

### 5-1. 地図でのピン表示・投稿
- OpenStreetMap 上にピンを表示。地図を**長押し**するとその場所にピンを立てられます。
- 上部に **NEED / OFFER / INFO の件数と緊急件数** を常時表示。
- 「現在地」ボタンで自分の位置へ移動、「ピンを立てる」ボタンで新規投稿。

### 5-2. 位置の指定方法
投稿時、位置は2通りで指定できます。
- **現在地から取得**（GPS）
- **地図で指定**：中央固定ピンを目的地に合わせる方式

### 5-3. 一覧表示・並べ替え
- ピンをカード形式で一覧表示。
- **新着順 / 緊急度順** で並べ替え可能。

### 5-4. 絞り込み（フィルター）
- **種別**（NEED/OFFER/INFO）、**ステータス**で絞り込み。
- 「対応済を隠す」オプション。
- モード（平時／災害）に応じた**おすすめフィルター**を自動適用。

### 5-5. 平時モード / 災害モード
- **災害モード**では、SOS・安否・物資ニーズを優先的に表示（上部に赤い警告バナー）。
- モードは設定画面から切り替え、端末に保存されます。

## 6. プライバシー保護：位置ぼかし機能

在宅避難者の安全に配慮した、SafePin の重要な特徴です。

- **NEED / OFFER のピン**は自宅など生活拠点と結びつきやすいため、
  正確な座標ではなく**約150m四方のマスの中心**に丸めて表示します（防犯上のリスク低減）。
- 地図上には「このあたり」を示す**円**を描画し、正確な地点ではないことを利用者に伝えます。
- **INFO（給水所・通行止め等）**は地点そのものが有用なため、正確な位置で表示します。
- 元データ（Firestore 上の座標）は変更せず、**表示時のみ**丸める設計です。

---

## 7. コミュニティ運用機能（ステップB）

住民運用での荒れ・混乱を防ぐための仕組みです。

### 7-1. ① 通報・自動非表示
- 不適切な投稿は誰でも「通報」できます。
- **3件**の通報が集まった投稿は**自動的に非表示**になります（一般ユーザーには見えなくなる）。
- 管理者は非表示の投稿も確認でき、必要に応じて**非表示を解除**できます。

### 7-2. ② 管理者モード（合言葉方式）
- 自治会役員など運用者は、**合言葉**を入力することで管理者モードに切り替えられます。
- 管理者は、**すべての投稿の編集・削除・ステータス変更・非表示解除**が可能。
- 合言葉は端末に保存され、設定画面からいつでも解除できます。

### 7-3. ④ 投稿の信頼度シグナル
投稿に対して住民が反応を付けられます。

| 反応 | 意味 | 効果 |
|------|------|------|
| 🟠 現地確認済 | 実際に見て確認した | **2人**が押すと未確認→現地確認済へ自動昇格 |
| 🟢 役に立った | 有用な情報だった | 信頼度の指標 |
| 🟤 古い情報 | もう古い可能性がある | 「役に立った」を上回り**3件**超で「古い可能性」警告を表示 |

- 一覧カードにも「古い可能性」バッジを表示し、鮮度を判断しやすくします。

## 8. サービス対象区域の制限（機能⑥）

地域限定の実証運用のため、投稿できる区域をガイドします。

- 対象区域は **下目黒4・5・6丁目**（e-Stat 令和2年国勢調査の丁目境界データを使用）。
- 地図上に**区域境界を緑のポリゴン**で表示。
- 投稿時、位置が**区域内か区域外か**をリアルタイムで判定・表示：
  - 区域内：緑のインジケーター／赤いピン
  - 区域外：オレンジの警告インジケーター／×付きオレンジピン
- 区域外でも**投稿はブロックせず「警告のみ」**（区域内投稿を推奨する運用）。

> ※ 国勢調査の丁目境界は、住居表示で用いる実際の境界と多少異なる場合があるため、
> あくまで目安として利用します。
> 出典：e-Stat 令和2年国勢調査 町丁・字等別境界データ

---

## 9. データ管理機能（設定画面）

- **投稿者名の設定**：次回以降の投稿にデフォルトで入る名前。
- **複数選択して削除**：自分の投稿（管理者は全投稿）をまとめて削除。
- **すべての投稿を削除**：自分の投稿（管理者は全投稿）を一括削除。
- **CSV / JSON インポート・エクスポート**：ピンデータを一括取り込み・書き出し
  （初期データ投入やバックアップに利用）。

---

## 10. 匿名性の仕組み

- ログイン不要。アプリ初回起動時に**端末ごとの匿名ID（UUID）**を自動発行し、端末内に保存します。
- このIDにより、「自分が投稿したピンかどうか」を判定し、**本人だけが自分の投稿を編集・削除**できます。
- メールアドレスや個人アカウントは一切不要で、**気軽かつ匿名で参加**できます。

## 11. 技術構成（概要）

| 領域 | 採用技術 |
|------|----------|
| フレームワーク | Flutter 3.35.4 / Dart 3.9.2 |
| 地図 | flutter_map + OpenStreetMap（APIキー不要） |
| バックエンド | Firebase（Cloud Firestore：リアルタイム同期 / Storage：添付） |
| 状態管理 | Provider |
| ローカル保存 | Hive（文書DB）/ shared_preferences（設定） |
| 位置情報 | geolocator |
| 匿名認証 | 端末発行の自前 UUID（外部認証サービス不使用） |

- **リアルタイム同期**：Firestore により、誰かがピンを立てると他の端末にも即時反映。
- **オフライン耐性**：ローカル保存を併用し、通信が不安定な災害時にも動作。
